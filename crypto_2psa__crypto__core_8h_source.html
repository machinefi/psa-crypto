35  * Number of locks on the key slot held by the library.
36  *
37  * This counter is incremented by one each time a library function
38  * retrieves through one of the dedicated internal API a pointer to the
39  * key slot.
40  *
41  * This counter is decremented by one each time a library function stops
42  * accessing the key slot and states it by calling the
43  * psa_unlock_key_slot() API.
44  *
45  * This counter is used to prevent resetting the key slot while the library
46  * may access it. For example, such control is needed in the following
47  * scenarios:
48  * . In case of key slot starvation, all key slots contain the description
49  * of a key, and the library asks for the description of a persistent
50  * key not present in the key slots, the key slots currently accessed by
51  * the library cannot be reclaimed to free a key slot to load the
52  * persistent key.
53  * . In case of a multi-threaded application where one thread asks to close
54  * or purge or destroy a key while it is in used by the library through
55  * another thread.
56  */
